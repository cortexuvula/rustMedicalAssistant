# v0.10.10 — Auto-Pipeline Carries Structured Patient Context Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make the auto-pipeline path honor structured patient context. Today, structured medications/allergies/conditions only flow through the manual `generate_soap` flow (v0.10.6); the auto-pipeline (`process_recording`) silently drops them. This release threads `patient_context` through `pipeline.launch` → `processRecording` → `process_recording` → existing `generate_soap`, mirrors the GenerateTab structured-context panel onto RecordTab so clinicians can fill the fields before/during recording, and validates oversized payloads before transcription consumes time.

**Architecture:** Five tasks. Task 1 extracts the existing `buildPatientContext` closure from GenerateTab into a shared `src/lib/utils/patient_context.ts` helper with vitest tests, refactoring GenerateTab to use it (DRY win, makes the helper testable in isolation). Task 2 extends the frontend pipeline plumbing (`stores/pipeline.ts` + `api/pipeline.ts`) with an optional `patientContext` parameter, backward-compatible. Task 3 extends the backend (`process_recording` Tauri command in `commands/pipeline.rs`) to accept `patient_context`, validate at entry, and thread to the existing `generate_soap` call. Task 4 wires RecordTab — three textareas + `$derived` Active badge + `buildPatientContext()` at every pipeline-launch site. Task 5 bumps the version to `0.10.10`.

**Tech Stack:** Svelte 5 (runes), TypeScript, vitest, Tauri 2, Rust (tokio, serde).

**Spec:** `docs/superpowers/specs/2026-05-01-auto-pipeline-patient-context-design.md`

### Step 3: Create the helper

Create `src/lib/utils/patient_context.ts`:

```typescript
import type { PatientContext } from '../types';
import { splitLines } from './text';

/**
 * Build a `PatientContext` payload from three line-per-item textarea values.
 * Returns `undefined` when every list is empty so the backend stores nothing
 * and renders no Patient record block.
 */
export function buildPatientContext(
  medicationsText: string,
  allergiesText: string,
  conditionsText: string,
): PatientContext | undefined {
  const medications = splitLines(medicationsText);
  const allergies = splitLines(allergiesText);
  const conditions = splitLines(conditionsText);
  if (medications.length === 0 && allergies.length === 0 && conditions.length === 0) {
    return undefined;
  }
  return {
    patient_name: null,
    prior_soap_notes: [],
    medications,
    allergies,
    conditions,
  };
}
```
