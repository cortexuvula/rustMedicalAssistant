# v0.10.12 — Split `RecordTab.svelte` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Decompose the 964-LOC `RecordTab.svelte` monolith into a thin orchestrating parent (~280 LOC) and three per-region child components in a new `src/lib/pages/record/` subdirectory. Behavior is preserved exactly — this is a maintenance refactor, invisible to users.

**Architecture:** Four tasks, smallest-first to prove the pattern. RecordingStateCards (pure presentation) → PipelineStatus (with `$effect` ticker + `bind:copyStatus`) → PatientContextPanel (largest, owns save-template modal) → version bump. Data flow uses Svelte 5 `$bindable` for state the parent must access; callback props for actions the parent owns.

**Tech Stack:** Svelte 5 (runes, `$bindable`, `$effect`), TypeScript, vitest.

**Spec:** `docs/superpowers/specs/2026-05-02-split-record-tab-design.md`

## Shared extraction recipe (applies to Tasks 1–3)

Each extraction task follows this pattern. The plan body for each task lists which **specific** state vars, handlers, props, and markup region to move.

### Recipe

1. **Read the parent file** (`src/lib/pages/RecordTab.svelte`) and identify the current line range of the region's markup, the section's state declarations, and any handlers that should move with it.

2. **Create the new component** at `src/lib/pages/record/<Name>.svelte` with this skeleton:

```svelte
<script lang="ts">
  // Imports (only what this child actually needs).
  // type Props with bindable + non-bindable + callback props.
  let { ... }: Props = $props();

  // Internal state (moved from parent if owned by this region).
  // Internal $effect / $derived (if any).
  // Internal handlers (moved from parent if owned by this region).
</script>

<!-- Markup moved from parent's region, verbatim. -->

<style>
  /* Scoped CSS rules moved from parent that style only this region's classes. */
</style>
```

3. **Move state, handlers, and markup** from the parent into the new file. Verbatim. Do NOT change behavior, naming, or logic.

4. **Move scoped CSS rules** that style ONLY this region's elements. For each rule in the parent's `<style>` block, ask: "Does this selector match a class used only inside this region?" If yes, move. If shared (e.g., `.btn-primary`, `.btn-secondary`, `.spinner`, `.error-text`, `.field-label`), leave in parent. If genuinely unclear, leave in parent and add `/* TODO: classify */`.

5. **Update the parent**:
   - Add the import: `import <Name> from './record/<Name>.svelte';`
   - Replace the inline markup with `<<Name> ... />` passing the appropriate props (bindable values via `bind:`, callbacks via `={handler}`).
   - Remove moved state, handlers, internal helpers, unused imports, scoped CSS rules.

6. **Verify and commit**:

{:else if $audio.state === 'recording'}
