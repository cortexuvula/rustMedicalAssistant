# v0.10.9 — In-Flight STT Cancellation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make pipeline cancellation actually interrupt an in-flight transcription. Today the cancel flag is checked at stage boundaries, so a user who clicks "Cancel" mid-STT still waits for the full HTTP timeout (or the local whisper pass) to finish. This release converts the pipeline cancellation primitive from `Arc<AtomicBool>` to `tokio_util::sync::CancellationToken` and threads it through the `SttProvider::transcribe` trait so HTTP futures (reqwest) drop on cancel and local providers exit promptly.

**Architecture:** Four tasks. Task 1 mechanically converts the cancellation type across the call chain (registry, command-layer signatures, trait method, both impls) with no behavior change — the workspace must compile and all tests pass at the end of Task 1. Task 2 wires real cancellation into `RemoteSttProvider::transcribe` via `tokio::select!` and adds a wiremock-backed integration test that proves cancellation interrupts an in-flight HTTP call within tens of ms. Task 3 wires the lighter-touch pre/post checks into `LocalSttProvider::transcribe` (the inner whisper call runs in `spawn_blocking` and is not interruptible without plumbing into whisper-rs callbacks, which is out of scope; the immediate-cancel-before-whisper case is what gets tested). Task 4 bumps the version to `0.10.9`.

**Tech Stack:** Rust (tokio, tokio_util, reqwest, async_trait), `wiremock = "0.6"` (already a dev-dep in `crates/stt-providers/Cargo.toml`).

**Audit reference:** Sprint 2 item B1 (cancellation through `SttProvider` trait); design choice A from this conversation (convert `AtomicBool` registry to `CancellationToken`, do not bridge).

### Step 4: Wire `tokio::select!` into the request path

```rust
let response = tokio::select! {
    biased;
    _ = cancel.cancelled() => {
        return Err(AppError::Cancelled);
    }
    result = self.client.post(&url).multipart(form).send() => {
        result.map_err(/* existing error mapping */)?
    }

```

And similarly wrap the response-body parsing if it's a separate await:

```rust

        return Err(AppError::Cancelled);
    }
    result = response.json() => {
        result.map_err(/* existing error mapping */)?
    }
};
```

`biased;` makes the cancel branch checked first on each poll, so a cancellation that fires while the HTTP future is still pending will be observed promptly.

## Task 3: Wire pre/post checks into `LocalSttProvider::transcribe`

**Files:**
- Modify: `crates/stt-providers/src/local_provider.rs`

The local provider runs whisper-rs inside `tokio::task::spawn_blocking`. The blocking call is not interruptible without plumbing into whisper-rs callbacks (out of scope — would require runtime support and changes to whisper-rs bindings). The minimum-viable improvement: check `cancel.is_cancelled()` immediately on entry and immediately after `spawn_blocking` returns. This ensures a user who cancels *before* the blocking call starts gets an immediate cancel; once whisper is running, the user still has to wait for it to complete, but the next stage (vocabulary correction, DB persist) checks the token and bails.

This is a real but limited improvement; document the limitation in the doc-comment.
